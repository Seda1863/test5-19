1. Amaç ve Kapsam

Bu doküman, mdx_guarantee_letter modülünün fonksiyonel kullanımı, teknik mimarisi, muhasebe kayıt kurgusu, kur/limit algoritmaları, hata senaryoları ve işletim kabiliyetlerini kurumsal standartta açıklamak amacıyla hazırlanmıştır.

2. Sistem Kabiliyetleri

  • Teminat mektubu yaşam döngüsünün uçtan uca yönetimi (taslak, onay, aktif, iade, nakde çevirme).
  • Alınan/Verilen mektup tipine göre otomatik nazım hesap kaydı.
  • Komisyon, blokaj ve tazmin için otomatik finansal yevmiye oluşturma.
  • Kur tipi bazlı (Doviz Alis/Satis, Efektif Alis/Satis, Manuel) döviz dönüşümü.
  • Banka limitlerinin limit para birimi bazında canlı kullanım ve uygunluk kontrolü.
  • 7/15/30 gün hatırlatma activity + e-posta otomasyonu.
  • Raporlama (liste, pivot, QWeb PDF), izlenebilirlik (chatter), rol tabanlı güvenlik.

6. Kur ve Döviz Mantığı

6.1 Kayıt Üzerindeki Kur Alanları

  • currency_id: İşlem para birimi (zorunlu değil, boşsa şirket para birimi kullanılır).
  • currency_rate_type: Kur seçimi.
  • letter_currency_rate: 1 Şirket PB = X İşlem PB.
  • letter_currency_inverse_rate: 1 İşlem PB = X Şirket PB.

6.2 Kur Seçim Algoritması

# guarantee_letter.py
if rate_type == 'forexbuying':
    rate = 1 / rate_obj.forex_buying
elif rate_type == 'forexselling':
    rate = 1 / rate_obj.forex_selling
elif rate_type == 'banknotebuying':
    rate = 1 / rate_obj.banknote_buying
elif rate_type == 'banknoteselling':
    rate = 1 / rate_obj.banknote_selling
elif rate_type == 'manualexchange':
    rate = manual_rate

inverse_rate = 1 / rate

6.3 Yevmiye Dönüşüm Algoritması

# balance (sirket para birimi)
balance_amount = company_currency.round(amount * inverse_rate)

# doviz izleme alanlari
line.currency_id = transaction_currency
line.amount_currency = +amount / -amount
TRY seçildiğinde dönüşüm 1.0 kabul edilir; ekran bozulmadan normal muhasebe akışı devam eder.

7. Banka Limit Mantığı

7.1 Canlı Kullanım Hesabı

bank.guarantee.limit üzerindeki kullanılan/kalan tutar, aktif ve iade talep durumundaki mektupların her birini limit para birimine çevirerek hesaplanır. 

# bank_guarantee_limit.py
for letter in active_letters:
    used += letter.currency_id._convert(
        letter.amount,
        limit.currency_id,
        limit.company_id,
        letter.issue_date
    )
limit.available_amount = limit.limit_amount - used

7.2 Onayda Limit Kontrolü

Onay anında talep edilen mektup tutarı limit para birimine çevrilir ve kalan limit ile karşılaştırılır.

# guarantee_letter.py::_check_bank_limit
requested = letter_currency._convert(
    self.amount,
    limit.currency_id,
    self.company_id,
    self.issue_date
)
if limit.available_amount < requested:
    raise UserError("yeterli limit yok")
Örnek: Limit = 900.000 TRY, mektup = 900.000 EUR. Sistem EUR tutarı TRY'ye çevirip kontrol eder; limit aşımı varsa onaya izin vermez.

8. Muhasebe Kurgusu

8.1 Nazım Kayıtlar

  • Alınan mektup: 900 borç / 901 alacak
  • Verilen mektup: 903 borç / 902 alacak
  • İade veya nakde çevirme: ters nazım kayıt

8.2 Finansal Kayıtlar

  • Komisyon: 780 / 102
  • Blokaj: 126 / 102
  • Tazmin (verilen): 689 / 300
  • Tazmin (alınan): 100 / 679

8.3 e-Dönüşüm Uyum Alanları

  • document_type = receipt
  • payment_method = bank
  • currency_rate_type, invoice_currency_rate, invoice_currency_inverse_rate
